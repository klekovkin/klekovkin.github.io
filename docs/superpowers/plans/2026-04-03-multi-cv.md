# Multi-Direction CV Pages — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create 6 CV page variants (3 career directions x 2 languages) from the existing single `index.html`, optimized for ATS processing.

**Architecture:** Six standalone self-contained HTML+CSS files. No build system, no JS (except a tiny toggle for military service). Each file is a complete page with embedded styles. The first file (`index.html`) is built from scratch as the Ukrainian Middle variant, then the other 5 are derived from it with content swaps.

**Tech Stack:** Pure HTML/CSS, Google Fonts (Roboto), inline SVG icons (Font Awesome paths)

---

## Files to Create/Modify

| File | Action | Responsibility |
|------|--------|----------------|
| `index.html` | Rewrite | UA Middle (blended) CV — the base template |
| `en.html` | Create | EN Middle (blended) CV — translation of index.html |
| `sre-ua.html` | Create | UA SRE/DevOps CV — technical emphasis |
| `sre-en.html` | Create | EN SRE/DevOps CV — translation of sre-ua.html |
| `manager-ua.html` | Create | UA PM/Manager CV — leadership emphasis |
| `manager-en.html` | Create | EN PM/Manager CV — translation of manager-ua.html |

---

## Reference: Shared CSS

All 6 pages use identical CSS. The full CSS block is defined once in Task 1 and reused verbatim in all subsequent tasks. The only addition to the current CSS is:

1. A `.lang-toggle` style for the language switcher in the sidebar
2. A `.military-service` collapsible style for the hidden-by-default military block (UA pages only)
3. A `.tech-skills` style for the technical skills list (used in SRE and Middle variants)

---

## Reference: Shared Sidebar HTML

The sidebar structure is nearly identical across all 6 pages. Only these parts change:

- `<html lang="...">` — `uk` for UA, `en` for EN
- `<title>` — name + tagline in the page language
- `<meta name="description">` — direction-specific keywords
- `.name` — `Юрій Клековкін` (UA) or `Iurii Klekovkin` (EN)
- `.tagline` — varies by direction and language (see spec)
- `.lang-toggle` links — point to the paired language file
- Section headings — Ukrainian or English
- Military service block — present only on UA pages
